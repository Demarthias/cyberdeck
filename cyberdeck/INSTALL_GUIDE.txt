INSTALLATION STEPS
------------------

1. COPY TO YOUR SYSTEM
   
   Transfer this entire "cyberdeck" directory to your target system
   (Termux, Linux server, or any bash-compatible environment).
   
   Suggested location: ~/cyberdeck

2. INSTALL DEPENDENCIES
   
   Termux:
   $ pkg install sqlite iproute2 bc -y
   
   Debian/Ubuntu:
   $ apt install sqlite3 iproute2 bc -y
   
   Arch:
   $ pacman -S sqlite iproute2 bc

3. RUN INSTALLER
   
   $ cd ~/cyberdeck
   $ chmod +x install.sh
   $ ./install.sh
   
   This will:
   - Create directory structure
   - Initialize database
   - Set permissions
   - Create control scripts

4. START THE SYSTEM
   
   $ cd ~/cyberdeck
   $ ./start.sh
   
   You should see:
   ✅ Cyberdeck started

5. VERIFY RUNNING
   
   $ bash healthcheck.sh
   
   All daemons should show: ✅ RUNNING

COCKPIT HUD (OPTIONAL)
═══════════════════════════════════════════════════════════════

Add to ~/.zshrc:

    source ~/cyberdeck/cockpit/cockpit.zsh
    PROMPT='$(cyberdeck_prompt) %n@%m %~ %# '

Then reload:

    source ~/.zshrc

Your prompt will now show threat status:
    ✓ = Green (safe)
    ⚡ = Yellow (suspicious)
    ⚠ = Red (blocked threat)

SYSTEM ARCHITECTURE
═══════════════════════════════════════════════════════════════

SENSOR → Monitors network connections
    ↓
INTELLIGENCE → Calculates threat scores
    ↓
    ├→ FIREWALL → Blocks malicious IPs
    ├→ CONTAINMENT → Quarantines processes
    └→ LOGGING → Records alerts

SUPERVISOR → Keeps all daemons running

CUSTOMIZATION
═══════════════════════════════════════════════════════════════

Edit config/cyberdeck.conf:

# More sensitive (detect more)
THREAT_THRESHOLD_NOTIFY=3
THREAT_THRESHOLD_BLOCK=6

# Less sensitive (fewer false positives)
THREAT_THRESHOLD_NOTIFY=7
THREAT_THRESHOLD_BLOCK=12

# Add trusted IPs
TRUSTED_IPS=("127.0.0.1" "192.168.1.100")

# Use iptables (requires root)
FIREWALL_MODE="IPTABLES"

Then restart:
    ./stop.sh && ./start.sh

═══════════════════════════════════════════════════════════════
TROUBLESHOOTING
═══════════════════════════════════════════════════════════════

DAEMON CRASHED?
    Check logs: cat logs/sensor.out
    Auto-restarts in 10 seconds

NO THREATS DETECTED?
    Lower thresholds: THREAT_THRESHOLD_NOTIFY=1
    Check connections: ss -tn | grep ESTAB

DATABASE LOCKED?
    Kill locks: fuser -k config/threats.db
    Restart: ./stop.sh && ./start.sh

HIGH CPU USAGE?
    Increase intervals in config:
        SENSOR_INTERVAL=10
        INTEL_INTERVAL=5
